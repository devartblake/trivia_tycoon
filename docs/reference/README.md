- Feature guides for completed features
- Architecture decisions from past work

## 🔍 How to Use This Directory

**Don't start here.** Check the main documentation directories first:
- `/phases/` - Current development phases
- `/api/` - API documentation
- `/implementation/` - Implementation guides
- `/architecture/` - Design decisions
- `/security/` - Security audits
- `/progress/` - Session summaries

Only refer to reference documentation when:
1. Looking for historical context or decisions
2. Understanding legacy implementations
3. Reviewing completed phases
4. Finding patterns from previous work
